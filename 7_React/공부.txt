# 핵심 개념

데이터(상태)가 어디에 있어야 깔끔한 구조가 되는지 + 컴포넌트끼리 어떻게 소통하는지

리액트 앱은 보통 아래와 같음
App(최상위)
- header
- TodoList
  - TodoItem
-Footer

여기서 생기는 고민
1. 어떤 컴포넌트가 어떤 상태(state)를 가져야 할까 ?
2. 어떤 컴포넌트가 어떤 UI만 담당하는 껍데기가 되게 할까 ?
3. 부모 -> 자식으로는 props로 값을 내려보내고,
   자식 -> 부모로는 props로 받은 "함수"를 호출해서 올려보낸다.
이게 바로

상태 끌어올리기 (Lifting State Up)
부모 ↔ 자식 데이터 흐름
props로 함수 전달 (onAdd, onChange, onDelete… 패턴)
재사용 가능한 UI 컴포넌트 만들기
이 네 가지로 연결되는 거야.
